# Governance Walkthrough

## Objective

Show how a draft moves through policy, evaluation, repair, and approval with evidence retention.

## Walkthrough

1. Ingest source material via upload or connector-controlled source.
2. Extract policy/terminology/scenarios with source evidence anchors.
3. Run draft evaluation and produce explainable score breakdown.
4. Apply repair suggestions with traceable before/after reasoning.
5. Enforce approval policy and auto-send guardrails.
6. Record audit events and send-event ledger entries.

## Controls Demonstrated

- Role-gated actions for sensitive operations.
- Runtime feature-flag controls and kill-switch posture.
- Legal hold/deletion safeguards.
- Trace IDs and auditable event streams.

## Output Artifacts

- Readiness board snapshot
- Audit events export
- Connector health/sync evidence
- Approval/send-event detail evidence
